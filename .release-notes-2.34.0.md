## 新增
- **AI Prompt 模板化**:用户可在设置里自定义 AI 摘要/预测的 prompt
  - 覆盖 3 个 AI 功能:📰 IT之家文章摘要 / 🏆 世界杯赛前预测 / 🏆 世界杯赛后总结
  - 每个 prompt 拆**角色设定 (system) + 输出规则 (rules)** 两字段,可独立编辑
  - debounce 500ms 自动保存 + toast 反馈
  - **默认值 = 此前硬编码原值,零行为变化**;用户改了才覆盖
- **Prompt 注册中心** (`prompt-registry.js`):3 个 prompt 的默认值 + `resolvePrompt`
  统一入口,为后续 A2 "该不该升级"建议铺路(加第 4 个 prompt 即可)
- **持久化**:`state.json.ai_prompts` 存用户自定义 prompt,PRESERVE_FIELDS forward compat

## 优化
- 零(本次纯加配置化能力,article-ai/match-ai 改用 resolvePrompt 但默认值不变)

## 技术细节
- **DEFAULT_PROMPTS 默认值 = 当前硬编码原值**(从 article-ai.js / match-ai.js 一字不差搬来)。
  article-ai (6) + match-ai (2) 现有测试全绿 = 零行为变化的硬验收。
- **resolvePrompt 整体替换语义**:用户配了某 prompt 且 system 非空 → 整体用用户配置;
  system 为空 → 回退默认。不做 system/rules 分别 fallback(避免混搭)。
- **worldcup 三段 prompt 统一**:角色句 + 内容指引并入 system,OUTPUT_RULES = rules。
- **vi.hoisted 测试技巧**:PromptSettings 测试用 vi.hoisted 创建 signal,
  绕过 vitest mock factory 的 hoisting 限制。
