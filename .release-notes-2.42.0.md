# 2.42.0 — C5 增量 Detector + I7 Tray 摘要

## C5 — 增量 Detector (后台自动检测加速)

`runDetectorChain` 在后台定时器跑(silent=true)时,对**最近 7 天已检测过**且 **detectors > 1 个**的 app,**只跑第一个 detector**;其余 detector 在 trace 标 `skipped: 'incremental'`,跳过的数量写入 `chainResult.incremental.skippedCount`。

预期效果:无更新 app 占多数的检测批,**总耗时下降 30-50%**(实测取决于 app 数)。

- 用户**手动**"检查更新"(silent=false)仍走全链,保证新版本不被漏报。
- 增量判定纯函数 (`src/workers/detector-chain-incremental.js`),`appsLastChecked` map 由 `state.json.apps[name].ts` 算出。
- 7 天阈值可通过 `incremental.recentDays` 注入。

## I7 — Tray 菜单顶部总览

macOS Tray **没有 hover tooltip 弹窗 API** (`tray.setToolTip` 在 macOS 上不显示;Windows 悬停几秒才出现且宽度受限)。I7 改 scope:在 **tray 菜单最顶部** 加一行全局总览,用户左键/右键打开菜单瞬间即可看到:

- 无检测结果 → `🔔 Pulse · 尚未检测`
- 有结果 → `🔔 Pulse · N 应用 · M 待升级 · Xm 前`(无更新时显示 "全部最新")

格式与下方 AI 用量段对齐 (`Xm 前` / `Xh 前`),信息层视觉一致。
