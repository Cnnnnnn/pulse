## 新增
- **IT之家未读角标**:SideNav 的「📰 IT 新闻」item 右上角显示红色数字胶囊,
  数字 = 本 session 新增且还没读的文章数
  - 读一篇 (`markIthomeRead`) → 数字自动 -1
  - 切到其他日期 / 重启 app → 归 0 (跟既有 `ithomeNewIds` 语义一致)
  - 折叠态 (40px 窄栏) / 展开态 (180px) 两套定位都适配
- **SideNav badge 扩展点**:`SideNavItem` 加了可选 `badge` prop,
  为未来其他面板 (wechat-hot 热搜等) 接未读角标铺路

## 优化
- 零 (本次纯加功能)

## 优雅降级
- ithome tab 未 bootstrap 时 `ithomeNewIds = {}` → `ithomeUnreadBadge = 0`
  → badge 不渲染 (符合预期)
- `ithomeUnreadBadge` 是纯 `computed` signal,无独立状态,无损坏风险

## 技术细节
- **`.side-nav-button` 补 `position: relative`**:现有 `side-nav-setup-dot`
  的绝对定位是相对整个 `.side-nav` (全侧边栏只有一个 dot 所以成立);本功能的
  badge 是**每 item 独立一个**,必须相对 item 自己的 button 定位,否则会错位。
- **`pointer-events: none`**:胶囊纯展示,点击穿透到下层 button 切 tab。
- 实现中发现 happy-dom 跨 test 共享 DOM,测试补了 `beforeEach` 清 body
  (TDD 实测暴露, plan 原漏此细节)。

## 风险

| 风险 | 等级 | 缓解 |
| --- | --- | --- |
| 折叠态 badge 与 setup-dot 视觉重叠 | 无 | ithome nav 不走 setup-dot (仅 AI 配置按钮),无交集 |
| 未读数超大 (>99) 胶囊溢出 | 极低 | 未读数实际很少超 9; `min-width` + `padding` 自适应;真超大属异常数据 |
| `computed` signal 被提前 GC | 低 | SideNav 持续订阅 (`void ithomeUnreadBadge.value`),生命周期=组件 |
