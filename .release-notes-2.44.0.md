# 2.44.0 — I6 v2 Wechat-Hot SideNav 角标清零

## I6 v2 — Wechat-Hot 角标自动清零

roadmap §13.4 推荐 wechat-hot badge 是 I6 v2 唯一未做的项, 标"需给 wechat-hot 从零补 read 概念(仿 ithome)"。

**对账(2026-06-24)**:I6 v2 wechat-hot 基建 **100% 已落地** — read-store / IPC / 推送 / signal / SideNav 集成 / 单测 全部齐全。**唯一缺口**:`setActiveNav('wechat-hot')` 时没有清未读角标的逻辑,导致 wechatHotNewIds 只通过用户点行 markWechatHotRead 单条消除,从没批量清零。对照 `funds` / `ai-usage` 的 `clearFundNavBadge` / `clearAiUsageNavBadge` 行为不一致。

### 改动

- `src/renderer/wechat-hot/store.js`: 新增 `clearWechatHotUnreadBadge()` — 重置 `wechatHotNewIds.value = {}` (session 级, 不动持久化的 `wechatHotReadIds`)。
- `src/renderer/worldcup/navStore.js`: `setActiveNav('wechat-hot')` (且 prev !== 'wechat-hot') 时调 `clearWechatHotUnreadBadge`, 行为对齐 funds / ai-usage。

### 行为

- 用户**首次切到 wechat-hot tab** → 角标自动清零 (跟 funds / ai-usage 一致)
- 用户**已在 wechat-hot tab** → 重复点不动 newIds (幂等)
- 切到其他 tab → 不动 wechat-hot newIds
- 持久化的 `wechatHotReadIds` **不被清** (用户已读的热搜词历史保留)
- 重启后 wechatHotNewIds 本来就归 0, 行为可观察
