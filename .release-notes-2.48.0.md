# v2.48.0 — 🔍 个股 AI 分析 (阶段四)

> 个股分析 tab: 输入 6 位股票代码 (带自动补全), 多选 7 个分析角度 (价格趋势 / 交易热度 / 估值水位 / 盈利能力 / 资金流向 / 技术指标 / 新闻舆情), AI 按真实数据客观解读, 不出具买入/卖出等投资建议.
> 60s 内存缓存 + 24h 持久缓存, 数据源 eastmoney 优先 → sina/tencent 备援, 单个角度失败不影响其他角度.

## 新增

- **🔍 个股分析 sidenav tab** — 侧边栏新增入口, 点开进入个股分析容器
- **🔍 个股 AI 分析抽屉** — 560px 右侧抽屉 (fade-only 动画)
  - 顶部股票代码输入框, 250ms 防抖搜索, 8 条候选下拉 (代码 / 名称 / 行业)
  - 7 个 angle chip 多选, 切到新 chip 立即 lazy 拉取该角度数据 (chip 标 ⚠ 表示拉取失败)
  - "🚀 开始 AI 分析" 按钮调 LLM 生成 `summary / perAngle / risks / signal` 结构化 JSON
  - 抽屉内显示: 总结 / 各角度解读 / 关注点 / 信号色标 (positive 绿 / neutral / cautious 橙)
  - 抽屉外点击关闭 (排除 `.stock-detail-open` 自身按钮), 复用 `BareModalShell` + cardRef 模式
- **7 个分析角度 fetcher** — 每个角度独立模块, 复用 eastmoney → sina 备援
  - `price_trend` — K 线近 30 日, 算 `change5d / change20d / amplitude`
  - `volume_turnover` — 算 `latestAmount / avgAmount30d / latestTurnover / avgTurnover30d`
  - `valuation` — F10 + tencent 备援, 算 `pe / pb / pePercentile3y` (F10 不给序列时降级 null)
  - `profitability` — F10 + sina HTML 正则备援, 算 `roe / grossMargin / netMargin / reportDate`
  - `capital_flow` — 东财 ffkline, 算 `mainNetInflow5d / mainNetInflow10d`
  - `tech_indicators` — 客户端纯算 `ma5 / ma10 / ma20 / macdHist` (EMA 序列 + 信号线)
  - `news_buzz` — 东财 np-listapi + sina feed.mix 备援, 取近 7 日新闻 + 客户端词典情感打标
- **失败隔离** — 7 个 fetcher 并行 (`Promise.allSettled`), 单角度失败只标该 chip ⚠, 不阻断其他角度或 AI 解读
- **24h 持久化缓存** — `state.stockDetailCache` 走 `PRESERVE_FIELDS`, 同 code + 同 angles + 同 dataHash 24h 内不重复打 LLM
- **60s 内存缓存** — IPC handler 层 `_detailCache` 60s TTL, 同 code + angles 60s 内不重拉数据
- **LLM 合规** — 复用 v2.47.1 的 P71 预算 + 投资建议关键词改写 (`买入/卖出/加仓/减仓/看多/看空/必涨/必跌/强烈推荐` → "当前市场呈现")
- **PII 防护** — 解析层加固 `userId|watchlist|searchHistory|selfSelect` 字段 → `[REDACTED]`
