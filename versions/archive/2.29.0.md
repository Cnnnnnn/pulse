## Pulse v2.29.0 (macOS · 诊断面板补全 — 启动时间 / 性能趋势 / Top-5 / 导出)

Q1 路线图第二阶段。诊断抽屉从「只看错误」升级为「启动 + 性能 +
Top-5 失败聚合 + 一键导出」四件套,出问题时能直接给 QA / 用户一个
`~/Desktop/pulse-diagnostics-{ts}.tar.gz` 复现包。

### 新增

- **⏱ 启动里程碑 (`src/main/diagnostics.js`)**：
  - 模块加载即拿 `t0` (`Date.now()`);后续调 `markBootstrapDone()` /
    `markRendererReady()` 记录两个里程碑
  - `markRendererReady()` 顺手把 `{ ts, readyMs }` 落 `state.json` 的
    `startup_samples` (cap 20,unshift 保持倒序),`PRESERVE_FIELDS` 加
    `kind: "array"` 跨 saveAll 不丢
  - 幂等:同一里程碑二次调直接 return,防止重复 unshift
  - `getStartup()` 返 `{ bootstrapMs, readyMs }`
- **📊 内存 / CPU 采样器 (同模块)**：
  - `startMetricsSampler(intervalMs=5000)` 起 5s 一个 `setManagedInterval`
    走 `timer-registry`,app quit 时随主 loop 清掉,不会泄漏
  - 每点 = `{ ts, heapUsed, rss, cpuUser }`,ring buffer cap 60
  - `getMetricsSummary()` 返 `{ latest, peak, count }`,peak = 期间最大
    heap / rss
- **🧮 Top-N 失败聚合 (`src/main/diagnostics-aggregator.js`)**：
  - `computeTopFailures(entries, topN=5)` 纯函数,按 `(source, message)`
    桶化,count desc → 最近 ts desc,缺字段走 `unknown/(no message)`
    兜底
- **📦 .tar.gz 诊断包导出 (同模块)**：
  - `bundleDiagnostics({ logsDir, extraLogsDirs, aggregator, outputDir,
    sinceMs, extras })` 把 `errors-*.jsonl` + 错误聚合器 top-10 + main
    日志 (`~/Library/Logs/AppUpdateChecker/startup.log` /
    `detect.log`) + diagnostics 当前快照 → `<outputDir>/pulse-diagnostics-
    {ts}.tar.gz`
  - outputDir 默认 `~/Desktop`,落地后 OS 自带 Archive Utility 直接打开
  - 用 POSIX ustar (512 字节块 + checksum) + `zlib.gzip`,**零第三方依赖**
- **🔌 主进程 IPC (`src/main/ipc/register-core.js`)**：
  - `diagnostics:fetch(opts)` 一次返 `{ startup, metrics, topFailures,
    stats, sinceMs }`
  - `diagnostics:fetch-samples()` 返 ring buffer
  - `error:export-zip` 从「返回 logsDir stub」升级为真导出,error-init
    aggregator 未装时也走 null-aggregator 路径,IPC 返 `{ ok, path,
    sizeBytes, fileCount }`
- **🪟 渲染层 (`preload.js` + `api.js`)**：
  - `diagnosticsFetch(opts)` / `diagnosticsFetchSamples()` 加入
    `window.api`
- **🖼 诊断抽屉 UI (`DiagnosticsDrawer.jsx` + `styles.css`)**：
  - 4 个新区块按顺序叠加:启动时间 → 性能 (heap / rss / cpu + 60 帧
    文本柱状 trend) → Top 5 failures → 导出按钮 + 结果回显
  - 导出按钮异步,导出中禁用,完成后显示 `.tar.gz` 路径 + size +
    fileCount,失败显示红色错误
  - 新增 `diag-section / __title / __loading / __empty`、`diag-row`
    (label + value)、`diag-trend` (bar graph)、`diag-failure`
    (count × source [main] message)、`diag-export__ok / __err` 样式
