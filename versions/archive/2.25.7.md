## Pulse v2.25.7 (Windows · 菜单栏配置 + 主面板 nav 联动)

新增 tray 菜单栏配置能力 + 主面板 nav tab 联动,用户可以独立开关 6 个 tray 菜单项,4 个动态段同时联动 SideNav 可见性。

### 新增

- **🧩 菜单栏配置 (tray menu prefs)**：tray 菜单底部新增「菜单栏配置...」入口,点开主面板内的 modal。配置 6 个项:
  - 4 个动态段(🔄 检查更新 / 📊 AI 用量 / ⚽ 世界杯 / 💎 贵金属)
  - 2 个底部 action(检查更新按钮 / 打开配置文件)
  - 「打开面板」「退出」锁死不可关 —— 避免用户把自己卡死
  - 持久化到 `state.json.tray_menu_prefs`,老用户升级自动全开
  - 模块边界:`src/main/tray-menu-prefs.js` 做 `TRAY_SEGMENTS` 单源真相 + `normalizePrefs` 纯函数
- **🔗 主面板 nav 联动 (v1 扩展)**：4 个动态段 prefs 同时联动 SideNav 对应 nav tab(`versions` / `ai-usage` / `worldcup` / `metals`)。3 个固定 nav(ithome / wechat-hot / funds)+ AI 配置按钮始终显示。当前 `activeNav` 被关时,`navStore.installNavWatch()` effect 立即切到第一个可见 nav —— 用户不会看到空白面板

### 修复

- 无 —— 纯功能新增,未触已有逻辑

### 改进

- **renderer 引入 `@main/*` alias**(`vitest.config.js` + `jsconfig.json`):让 renderer 也能 import `src/main/tray-menu-prefs.js`,避免 `TRAY_SEGMENTS` 在 renderer 测试里重复定义常量。生产 bundle 不受影响(esbuild 跟 vitest 走同套 alias 解析)
- **tray-init bridge 模块**(`src/main/bootstrap/tray-init.js`):消除 IPC handler 和 main bootstrap 之间的循环依赖 —— IPC handler 通过 getter 拿 `trayMgr`,bootstrap 在 install 后 setter 注入
