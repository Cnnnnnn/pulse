# v2.45.0 — A1/A2 UX 打磨

> 后续 §14 复盘后第一轮"主体已 100% 落地"项目打磨.
> 目标: 不动主流程 / prompt / cache key, 只提升 A1 changelog 摘要 + A2 升级建议的用户可见体验.

## 改动

- **新增 `src/ai/ai-errors.js`** — 错误 reason 中文化字典 (api_key_missing / llm_failed / timeout / parse_failed / app_not_found / no_update / invalid_args), 输出 `{ label, hint, raw }` 三段式
- **A1 ChangelogSummary 改造**:
  - loading 改骨架屏 (pulse 动画) + "AI 提炼中 · 通常 5–15s" 文案
  - 错误条改用 `humanizeAiError` 友好中文 + 保留 raw 详情到 title
  - 命中缓存时显示 "3m 前生成" 小字提示
  - oneLiner > 60 字 / highlights > 50 字自动截断 + …
- **A2 UpgradeAdvice 改造**:
  - reasons 列表渲染 (▸ 前缀), 不再透出 JSON 字符串
  - 命中缓存时显示 "3m 前生成" 小字提示
  - 错误条改用 `humanizeAiError` 友好中文
  - loading 文案 "💡 AI 分析中 · 通常 5–10s"
- **styles.css** — 加 `.changelog-summary-skel` 骨架 + pulse 动画 + `.changelog-summary-cached` / `.upgrade-advice-cached` / `.upgrade-advice-reasons` 样式
