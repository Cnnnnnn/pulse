# v2.47.1 — 📈 选股分析 (阶段二): AI 推荐策略 + 抽屉层级修复

> 选股筛选器新增 AI 推荐抽屉 — 描述意图 (低估值修复 / 高分红防御 / 超跌反弹 / 成长动量 / 行业龙头 / 平衡型) + 可选自由文本, 走 LLM 生成 criteria + sortConfig, 一键应用到条件区.
> 同时修复阶段一遗留下来的 2 个布局 bug.

## 新增

- **🧠 AI 推荐策略抽屉** — 在「选股」tab 头部点 `🧠 AI 推荐` 打开
  - 6 个预设 chip (低估值修复 / 高分红防御 / 超跌反弹 / 成长动量 / 行业龙头 / 平衡型) + 自由文本 (可补充行业 / 风险偏好 / 持有周期)
  - 点 `🚀 生成推荐` 走 LLM 生成 `criteria + sortConfig + 市场总结`
  - 24h 持久缓存 (state.json.aiStockAdviseCache), 同条件 24h 内不重复打 LLM
  - 预览面板可取消 / 应用; 应用后条件区自动填入, 排序立即本地重排
  - prompt 走 `resolvePrompt` 走 prompt-registry, 用户可在 AI 设置里修改
  - PII 安全: 不发 `userId / watchlist / searchHistory` 到 LLM
  - summary 合规改写: 命中"买入/卖出/加仓/减仓/看多/看空/必涨/必跌/强烈推荐"等投资建议关键词时整句替换为中性描述
  - 预算硬限: 复用 P71 token 预算, 超出返 `budget_exceeded`

## 修复

- **fix(stocks): 拆分 crypto 依赖** — `computeMarketOverview` + `medianOf` + `percentileOf` 从 `stock-constants.js` 拆到 `src/stocks/market-overview.js` (node-only).
  - 原因: `stock-constants.js` 被 renderer 端 (`stockStore.js` / `CriteriaPanel.jsx`) 引入, esbuild 默认浏览器平台找不到 node 内置的 `crypto`, 报 `Could not resolve "crypto"` 阻断 `npm start`.
  - 测试同步拆分到 `tests/stocks/market-overview.test.js`.
- **fix(stocks): AI 抽屉层级修复**
  - overlay 透明 + `pointer-events: none` — 不再以半透明黑蒙层盖住列表
  - drawer 改为 `fade-only` (无 `translateX` 滑入), 表格容器在抽屉打开时**立即**让出 360px + 16px `padding-right` — 消除"drawer 滑入过程中穿插覆盖列表表头"的视觉抖动感
  - 抽屉外点击关闭 (`document.mousedown`, 排除打开触发按钮 `.stock-btn-ai` 自身避免闪烁)
  - `ModalShell` / `BareModalShell` 新增 `cardRef` prop (默认 null, 不影响其它使用方)
