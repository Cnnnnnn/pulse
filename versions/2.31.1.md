## 新增
- **更新日志首启引导 (Onboarding)**:升级 Pulse 后,首次启动自动弹一个多步
  向导,展示本版本的更新日志 + 重点功能介绍
  - 第 1 页是完整的 changelog (走 marked + DOMPurify 安全渲染)
  - 第 2~N 页是重点功能介绍 (每页一个功能: 标题 + 副标题 + 正文)
  - 进度点 + 翻页 (← → 键盘可达),底部 `[跳过] [上一步] [下一步] [完成]`
  - 4 种关闭路径都算"看过本版":点跳过 / 翻到最后一页点完成 / 按 ESC / 点遮罩
- **随时重看**:Header 加 `📖` 按钮,点击可重新打开当前版本 release notes
  (走"手动"路径,**不**写已看标记 → 红点状态不受影响)
- **未读红点**:当前版本未看过时 `📖` 角标显示红点,看过后消失
- **持久化**:`state.json.last_seen_release = { version, at }` 记录最近看过
  的版本,跨重启保留 (PRESERVE_FIELDS 自动保留,老 state.json 兼容视为未看)

## 优雅降级
- 发版漏 `.release-notes-<ver>.md` → 不弹 + log warn (不阻断启动)
- `slides.json` 缺失 / parse 失败 / schema 不符 / 空数组 → 向导退化成单页
  changelog + 完成按钮
- `state.json` 损坏 → 视为未看,正常弹 (Q8 corruption recovery 兜底)
- `mark-seen` IPC 失败 → toast `保存失败, 下次启动还会再弹` + 向导仍正常关闭

## 范围说明 (v1)

- 严格限定:**只引导 Pulse 自身的 release notes**。
  `ChangelogPanel.jsx` 拉"被监控 app"的 release notes 是另一条管线,本次不动。
- 不做多语言 / 截图嵌入 / 全局开关 / 永久跳过 / 视觉庆祝动画 / 桌面通知。
- 老版本 (2.25~2.31) 历史 `.release-notes-*.md` 不补 slides.json,只有未来
  发版才会带 slides。

## 风险

| 风险 | 等级 | 缓解 |
| --- | --- | --- |
| 发版漏 `.release-notes-<ver>.md` | 中 | `getCurrent` 返回 `null` → 不弹 + log warn |
| state-store 写失败 | 低 | mark-seen 失败 → toast + log warn,下次启动会再弹 |
| 用户跳过 2.31.1 直接装 2.31.2 | 低 | 版本不一致 → 弹 2.31.2 的内容 (自然) |
| focus trap 没做 | 低 | ESC + ← → + Enter 覆盖基本键盘可达性,v2 再补 |
