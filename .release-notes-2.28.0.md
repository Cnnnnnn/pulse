## Pulse v2.28.0 (macOS · 升级前快照 + 一键回滚)

新增 App rollback 能力,brew 类应用升级前自动备份最近 2 个 .app 版本到本地,
升级后通过主面板的 ⏪ 按钮打开回滚历史,一键恢复到任一旧版本。

### 新增

- **📦 升级前自动备份 (`src/main/backup.js`)**：
  - 每次 brew upgrade 之前把当前 .app 复制到 `userData/backups/<BundleName>/<fromVersion>.app/`
  - cap 2:每个 app 最多保留最近 2 个备份,超出自动 prune 最旧的
  - cp 前 rm 防 merge (I-1 修复,跟旧 backup 一致):残留文件不会跟新 .app 撞车
  - dirSize 用纯 Node fs (lstat + readdir 递归),不 spawn `du`,sandbox 下不卡
- **🗂 升级历史 (`src/main/version-history.js` + state-store `version_history` 字段)**：
  - 扁平 map `{ appName: [ { from, to, at, backupPath, source, sizeBytes } ] }`,倒序
  - cap 2 在 state 层做,fs prune 在 backup 层做,职责单一
  - state.json 字段 PRESERVE_FIELDS 标注为 `notArray: true`,跨 patchState 安全
- **⏪ 一键回滚 (`src/main/rollback.js`)**：
  - 流程: 优雅退出 app (osascript) → 5s wait → pkill -9 兜底 → trash 旧 .app → cp 备份
  - in-flight Set 锁:同 app 第二次并发回滚返 `in_progress`
  - 回调 try/catch 包住:onUpdateInstalled/onActivity/onBroadcast throw 不影响 rollback 成功
  - trashItem 失败时 fallback 到 fs.rmSync (跟 I-1 一致)
- **🖥 主面板 UI (`src/renderer/components/VersionHistoryDrawer.jsx`)**：
  - 480px 右滑抽屉,每条 entry 显示 版本号 / 时间 / 大小
  - 两条 action:「回滚到这版」(主按钮) + 「删除」(清理磁盘)
  - 在打开时订阅 main 的 `version-history-updated` 广播,自动 refetch
  - 成功后自动关闭抽屉;失败时保持打开让用户看原因
- **🧮 ⏪ 角标 + 可见性 (`src/renderer/store-version-history-counts.js`)**：
  - AppRow 的 ⏪ 按钮旁显示当前 app 的可回滚版本数
  - main 每次 check / bulk-upgrade / rollback / delete 后 broadcast `version-history-counts-updated`
  - renderer store 装一次 listener,幂等

### IPC 新增 (4 通道)

| 通道 | 方向 | 用途 |
| --- | --- | --- |
| `get-version-history` | renderer → main | 拉单个 app 的 entries + 跨 app totalSize |
| `rollback-app` | renderer → main | 触发一次回滚,返 `{ok, reason?, error?}` |
| `delete-backup` | renderer → main | 删 fs 备份 + state entry,返 freed bytes |
| `version-history-updated` | main → renderer | 每次成功回滚后 broadcast,drawer 监听 refetch |
| `version-history-counts-updated` | main → renderer | 全量 count map,AppRow ⏪ 角标 |

### 修复

- **`backup.dirSize` 不用 `child_process.execFile` (`du -sk`)**：
  - 之前在 sandbox 测试环境会卡 8s 超时,实测会让整个测试进程看似 "卡死"
  - 重写为纯 `fs.promises.lstat` + `fs.promises.readdir` 递归,跨平台一致 + 零 spawn
  - 加静态回归保护: `tests/main/backup.test.js` 校验 `backup.js` 不再 `require('child_process')`
- **Vitest CJS/ESM interop 修复 (跟之前 backup hook 测试同源)**：
  - `tests/main/bulk-upgrade-with-backup.test.js` 改用 `createRequire(import.meta.url)` 拿 backup/version-history
  - `register-core.js` 把 `const { doRollback } = require("../rollback")` 改成 `const rollback = require("../rollback")`
    然后 `rollback.doRollback(...)`:解构导入会在 require 时把函数值复制,后续 `vi.spyOn` 改 module exports 不会反映到闭包
  - 这一类问题在 jest-free 环境下反复出,统一成 "CJS require 拿 module 对象,spy module.exports.X" 范式

### 改进

- **state-store 新增 helper `saveAppInstalledVersion(name, version)`**：只写 `apps[name].installed_version` 一条,不动 latest_version / has_update / changelog,留给下次 detect-run 自然刷新
- **rollback IPC handler 用 safeHandle 模式**:`logMeta` + `onError` 跟 `snooze:set` 一致
- **bulk-upgrade 完成时 broadcastVersionHistoryCounts 兜底**:`.then` 和 `.catch` 两条路径都推一次,保证 renderer 状态最终一致
