# 2.43.0 — A7 v3 Daily Digest LLM 改写

## A7 v3 — Daily Digest LLM 接入

roadmap §13.4 标 A7 为"补半成品"(`shared-llm.js` 全有,只差 `config.aiDigest.prompts` 配置化 + Settings 面板)。

**对账后实际状态**:A7 基建**远超预期** — `state-store.loadAiPrompts/saveAiPrompts` + `src/ai/prompt-registry.js` (DEFAULT_PROMPTS 6 个 key + `resolvePrompt(key)` 合并用户配置) + `register-ai-prompts.js` IPC (load/save/reset) + `prompt-store.js` signal + `PromptSettings.jsx` 完整 UI (system + rules + few-shot + 恢复默认) **都已落地**(在 v2.34 / v2.36 等历史版本)。

**唯一缺口**:`daily-summary-job.checkAndPush` 完全没接 LLM,硬编码 `result.lines` 直接 push。本版把这条补上。

### 改动

- `src/ai/prompt-registry.js`: DEFAULT_PROMPTS 新增 `daily_digest_summary` key(system + rules + few-shot),用户可在 `PromptSettings` UI 直接编辑。
- `src/main/digest/daily-summary-job.js`:
  - `checkAndPush` 变 async,新增可选步骤:push 之前调 `tryRewriteSummary(lines, date, deps)` 走 LLM 改写要点行。
  - `tryRewriteSummary` 纯函数:
    - 用 `resolvePrompt('daily_digest_summary')` 拿 prompt
    - 拼 messages = `[system, user(规则+日期+要点)]`
    - 调 `sharedLlm.chatCompletion(messages)`,8s `Promise.race` 超时
    - 成功且 text 非空 → split lines 返回新数组
    - 失败/超时/空 → **回退原 lines**(不破现有 push 行为)
- `triggerNow` 返回 Promise(测试 async 调用)。

### 行为

- 用户**没配置** AI provider / API key → 自动回退到硬编码模板字符串(原行为),零破坏。
- 用户**配置了** provider 且 LLM 调通 → 早报推送的 body 变成可读段落,例如原本的 `• Cursor 1 → 2` 改写成 `今日 Cursor 有新版\nKimi 还在升级`。
- LLM 调用超时(8s)或失败 → 回退原 lines,**用户感知不到区别**(静默降级)。

### UI

`PromptSettings` 面板(AISettingsModal 内)自动出现 **"每日早报改写"** 一项,沿用现有 system + rules + few-shot + 恢复默认 4 段式 UI,跟其他 prompt (ithome/worldcup/升级建议/changelog 摘要/分类) 风格一致。
