# 贵金属模块 UI 重新设计 · 设计分析

> **设计方**：UI Designer
> **日期**：2026-07-12
> **目标**：针对贵金属（黄金/白银，含国际 XAU/XAG 与国内 AU9999/AG9999）模块，从用户体验、信息架构、视觉、响应式、可实施性五个维度做重新设计分析，并给出方案对比与推荐。
> **前置约束**（来自现状代码与设计体系，不可越过）：
> - 数据层（scheduler / fetcher / IPC / preload）**不动**；renderer 只换皮与重组。
> - `--metals-*` 令牌当前已映射到应用统一语义令牌（`--color-up/--color-down/--accent-primary/--bg-card` 等），遵循 `ui-design-system.md`。
> - 技术栈 Preact + preact/hooks；CSS 令牌护栏（禁止裸 hex）。
> - 配色逻辑沿用 **A 股习惯：涨红跌绿**（`--color-up` 红 / `--color-down` 绿）。

> 📌 **范围修订（末次决策 · 2026-07-12）**：本模块**最终收敛为「纯行情数据看板」**——只做**行情追踪 + K线/指标 + 市场状态**展示，**既不提供任何交易下单能力，也不包含任何持仓/记账内容**（无下单票、无订单、无券商接入、无模拟/实盘交易；无持仓录入、无成本/盈亏/目标备注展示）。此前 §7 的「对接真实券商」「下单票」「订单状态机」等内容，以及「我的持仓（本地记账）」区块**均已作废**，相关 UI 已从原型与交付规范中移除。下文 IA / 路线图 / 组件均以纯行情看板为准。

## 0. 现状速写（基于代码，非假设）

当前模块 = Phase 4 收敛后的「**单层密集表格**」形态：

```
metals-layout (跟随应用主题，非独立深色)
├── metals-header  单行 status bar：标题 + 总市值/总盈亏/今日预估 + 刷新
└── metals-table   6 列 × 4 行 (XAU / XAG / AU9999 / AG9999)
    品种 | 最新价(¥/克) | 涨跌% | 30天sparkline | 持仓 | 操作(★/⋯)
```

- **行情**：仅「最新价 + 涨跌% + 30 天 sparkline」。统一折算成 **¥/克**（国际品种经 FX 汇率换算）。
- **持仓（现状，本次重设计移除）**：嵌在表格「持仓」列里（数量 + 累计盈亏）；空仓显示「+ 录入持仓」文字链，点开 `AddMetalModal`。重设计后此部分不再保留。
- **关注**：行内 ★ 按钮加入/移出 watchlist。
- **缺失**：无 K 线 / 分时、无独立持仓展示区、无市场总览、无可添加品种（下拉仅 4 个固定枚举）。
- `MetalTrendStrip` 已标「废弃」、`selectedMetalId` 信号无人驱动——说明此前的「走势详情」交互被砍掉，当前**没有任何 drill-down 入口**。

> 一句话定性：**它现在是一个「贵金属行情追踪」工具，不是「交易终端」，也不是「持仓/记账本」**。本次重设计**收敛为纯行情数据看板**——行情榜 + 选中品种 K线/指标 + 市场状态，明确**不含任何交易下单能力，也不含任何持仓/记账内容**。

## 1. 用户体验痛点识别

| # | 痛点 | 证据 / 现状 | 影响 |
|---|------|------------|------|
| P1 | **无交易操作闭环** | 只有「录入持仓」记账，没有任何买/卖意图表达、下单草稿或交易记录 | **本次明确不做交易**：模块定位纯行情数据看板，该痛点不在范围内；用户决策依赖行情展示即可 |
| P2 | **持仓管理被压扁（现状）** | 持仓盈亏挤在表格单列，无成本基础分解、无汇总分析、无单仓详情 | **本次明确不做持仓**：模块收敛为纯行情看板，该痛点不在范围内 |
| P3 | **行情可视化过浅** | 仅 30 天 sparkline；无 K 线（项目已有 `CandlestickChart.jsx` 却未用）、无分时、无高低开、无区间切换 | 看不懂趋势结构，决策依据弱 |
| P4 | **信息架构扁平单薄** | 全模块 = 一张表；行情 / 持仓 / 关注三个概念混在一行；「添加关注」与「录入持仓」在弹窗里二合一 | 概念混淆，扩展无位；新增能力只能继续往表里塞 |
| P5 | **空态与异常态弱** | FX 缺失时总览显示「—」；行级 error 才提示；无市场开/休市状态、无会话时间 | 数据断流时用户困惑，缺乏"现在行情是否可看"的语境 |
| P6 | **无 drill-down 入口** | `selectedMetalId` 已无 UI 驱动，`MetalTrendStrip` 废弃 | 点任意品种无法展开详情，表格即终点 |
| P7 | **可扩展品种为零** | `AddMetalModal` 品种下拉 = 固定 4 枚举；`watchedIds` 写死 | 铂钯、积存金、ETF 等无法纳入，模块天花板明显 |
| P8 | **交互反馈单薄** | 刷新仅转圈字符；保存后直接关弹窗无 toast；无乐观更新 | 操作缺乏确认感与状态可见性 |
| P9 | **红绿依赖颜色** | 涨跌仅靠 `--color-up/down` 着色，无 ▲▼ / +− 字形辅助 | 色觉障碍用户无法分辨涨跌 |
| P10 | **窄屏适配粗糙** | 仅 `<800px` 隐藏持仓列 + 压缩 sparkline；无卡片化降级 | 侧栏/窄窗下体验割裂，表格溢出 |

## 2. 信息架构优化方案

### 2.1 核心功能拆解

| 功能域 | 现状 | 期望（本次设计目标） |
|--------|------|----------------------|
| **行情展示** | 4 品种价格 + sparkline | 行情榜 + 选中品种详情（K 线/分时/关键指标）+ 市场上下文 |
| **交易操作** | 仅记账（现状） | **（本次不做）** 模块定位纯行情数据看板，不含下单 / 交易记录 / 券商接入；现状的「录入持仓」记账也一并移除 |
| **持仓管理** | 表格单列（现状） | **（本次不做）** 模块收敛为纯行情数据看板，不含任何持仓录入 / 成本盈亏 / 记账展示 |

### 2.2 三套 IA 候选方案对比

**方案 A — 维持单层表格（现状增强）**
```
[ status bar 总览 ]
[ 表格: 品种 | 价 | 涨跌 | 走势 | 持仓 | 操作 ]
```
- ✅ 改动最小、最贴合现有代码，风险低
- ❌ 无法承载行情详情/市场状态，P3/P4 不解决，扩展性差

**方案 B — 双栏：行情榜 + 上下文面板（推荐）**
```
┌──────────────────────────┬─────────────────────────────┐
│ Header: 标题 + 市场状态    │ 选中品种详情面板            │
│ ──────────────────────── │ ─────────────────────────── │
│ 行情榜 (可滚动表格/列表)   │ K线/分时 + 高低开 + 区间切换 │
│  XAU  ¥xxx   ↑x%  ▁▂▃     │ 关键指标卡 (52w高/低/波动)   │
│  XAG  ...                 │ 市场状态区 (交易中/休市)     │
│  AU9999 ...               │ （纯行情看板 · 无交易/持仓） │
│  AG9999 ...               │                             │
└──────────────────────────┴─────────────────────────────┘
```
- ✅ 引入 drill-down 解决 P6；行情与关注/市场状态分区，去除交易/持仓噪音解决 P4
- ✅ 纯前端重组，不动数据层；复用 `CandlestickChart`
- ⚠️ 右侧面板需响应式折叠（见 §4）

**方案 C — 多 Tab：行情 / 持仓 / 交易（完整形态）**
```
[ Tab: 行情 | 持仓 | 交易 ]
 行情: 榜 + 详情(同B右栏)
 持仓: 汇总卡 + 持仓列表 + 归因
 交易: （本期不做 · 纯数据看板）
```
- ✅ 功能最完整、概念最清晰
- ❌ Tab 切换割裂"看行情顺便看状态"的连贯流；占用更多导航；移动端 Tab 更挤
- ⚠️ 真实交易需 P2 后端，短期是"草稿/模拟"

### 2.3 推荐

> **采用方案 B 作为本次重设计主结构（右栏 = 纯行情详情：K线/指标/市场状态，不再含持仓区块）**。
> 理由：贵金属是"盯盘"场景，用户心流是「看行情榜 → 点某品种 → 看 K线/指标/市场状态」；横向双栏比纵向切 Tab 更顺，且在不引入新导航的前提下解决了 P4/P6，实施风险可控。C 的「交易 / 持仓」Tab 本期均不做（纯行情数据看板范围外），如需完整形态后续再评估。

**B 方案下的布局骨架（桌面 ≥1024px）**
```
metals-layout (grid: 行情榜 1fr | 详情面板 380–420px)
├── metals-header  (跨两栏): 标题 · 市场状态徽标 · 刷新
├── metals-watchlist (左)
│     └── 行情榜行 (可点选 → 驱动右侧 selectedMetalId；★ 行内关注切换)
└── metals-detail (右, 随选中品种更新)
      ├── 大图: K线/分时 + 区间(日/周/月) + 高低开量
      ├── 指标卡: 现价/涨跌/振幅/52w高/52w低
      ├── 市场状态区: 交易中/休市 + 会话时间
      └── （交易下单 / 持仓记账 不在范围 · 纯行情看板无此卡）
```

## 3. 视觉设计方向

### 3.1 色彩（金融产品调性）

原则：**先服从统一设计体系，再用"金"做克制的模块品牌暗示**（贵金属 ≠ 必须用金，但一抹金能强化品类认知，且不破坏全局蓝主色）。

| 方向 | 主色 | 强调/品牌 | 评估 |
|------|------|-----------|------|
| V1 纯跟随体系（现状） | 应用语义色 | `--accent-primary` 蓝 | ✅ 零新增令牌、最稳；但模块无辨识度 |
| **V2 蓝主色 + 香槟金点缀（推荐）** | 蓝（全局） | 选中态/品牌角标用香槟金 `--metals-gold` | ✅ 金融稳重 + 品类暗示；金仅用于焦点/选中/logo 角标，用量 <5% |
| V3 金色主导 | 金为主 | 红涨绿跌 | ❌ 偏离全局蓝、易显"土豪"、对比风险 |

- **涨红跌绿**：沿用 `--color-up`(红)/`--color-down`(绿)，且**必须叠加字形**（`▲/▼`、`+/−`）满足 P9 与 WCAG「不依赖颜色」。
- **建议新增一个 scoped 令牌** `--metals-gold: #C8A24B`（香槟金，已按对比度校验：浅底深字 ≥4.5:1，暗底亮金 ≥3:1 用于 UI 元素）；仅作用于 `.metals-layout` 子树，不污染其他模块（符合 §8 护栏）。

### 3.2 数据可视化方式

| 视图 | 现状 | 建议 | 优先级 |
|------|------|------|--------|
| 榜单迷你走势 | 30d sparkline (140×28) | 保留，加 ▲▼ 端点 + 涨绿跌红一致 | P0 |
| 选中品种主图 | 无 | **接入 `CandlestickChart`**：K线 + 区间切换(日/周/月) + 成交量副图 | P1 |
| 关键指标 | 无 | 指标卡：现价/涨跌/振幅/52w高/低/年化波动 | P1 |
| 持仓盈亏归因 | 单列数字（现状） | **（本次不做）** 纯行情看板不含持仓，故无归因卡 |
| 市场热度 | 无 | （P2）板块热力/相关品种联动 mini-map | P2 |

> 主图用项目已有的 `CandlestickChart.jsx`，避免重造；区间切换复用其 props。

### 3.3 交互反馈设计

- **刷新**：按钮内联 loading（禁用态 + spinner），完成给轻量 toast「已更新 HH:MM」；尊重 `prefers-reduced-motion`。
- **关注/取消关注**：★ 状态切换即时反馈 toast「已加入关注 / 已取消关注」，无需模态。
- **选中品种**：左榜选中行左侧 2px 金线 + 浅底高亮（非纯色块），右侧面板 150ms 淡入。
- **骨架屏**：冷启动沿用现有 `.metals-cell-skeleton`，扩展到详情面板（图区占位）。
- **空/异常态**：FX 缺失时行情区改为「汇率更新中」徽标而非「—」；整模块首屏若无数据给引导空态（非仅单行）。
- **市场状态**：header 加「交易中 / 休市」语义徽标（`--color-success`/`--text-tertiary`），给行情语境（P5）。

## 4. 响应式适配

应用是 Electron 桌面端（带 SideNav），但窗口可窄、也可投屏/平板，需分级：

| 断点 | 布局 | 关键降级 |
|------|------|----------|
| **≥1024px** | 双栏（行情榜 + 详情面板 380–420px） | 完整 K 线 + 指标卡并排 |
| **768–1023px** | 双栏但详情面板收窄至 320px；K 线/指标上下堆叠 | 区间切换保留 |
| **<768px（侧栏/窄窗）** | **单栏卡片化**：行情榜 → 点选展开为全宽详情（右栏变覆盖层/抽屉），或上下分屏 | 表格转为卡片行；sparkline 宽度 `min(100%, 160px)` 自适应；触控目标 ≥44px |
| **触控** | 行高/按钮热区 ≥44px；★/编辑按钮间距加大 | 避免 hover-only 操作 |

- 表格 → 卡片：窄屏下列改用 `flex` 卡片（品种头 + 价/涨跌 + 走势 + 关注星），复用同一数据源。
- 详情面板在窄屏以 `DrawerShell`（项目已有）覆盖，保持 drill-down 不丢。
- 所有图表容器 `width:100%` + `ResizeObserver`/CSS `aspect-ratio` 自适应，避免写死 140px。

## 5. 可实施性评估

### 5.1 分级路线图（按数据层依赖）

| 级别 | 内容 | 数据层依赖 | 风险 |
|------|------|-----------|------|
| **P0 纯前端重组** | 双栏布局、header 加市场状态、选中驱动右栏、红绿加字形、金点缀、空态优化、响应式卡片化 | 无（复用 `selectedMetalId` 信号 + `CandlestickChart`） | 低 |
| **P1 可视化增强** | K 线主图 + 区间切换、指标卡（现价/涨跌/振幅/52w高/低）、市场状态徽标 | 轻（历史 kline 已存 `historyMap`） | 中 |
| **P2 交易 / 持仓 / 扩展（本期不做）** | 真实下单 API、本地持仓记账、可添加品种（铂钯/ETF/积存金）、市场热力 | **重（需 fetcher/IPC/后端扩展，且超出"纯行情数据看板"定位）** | 高，需单独立项、另行评估 |

> 建议本次只交付 **P0 + P1（纯行情数据看板）**；**交易下单与持仓记账均明确不在范围内**，P2 单独评估，不阻塞本次重设计。

### 5.2 关键风险与护栏

| 风险 | 缓解 |
|------|------|
| 红涨绿跌与全局 `--color-up/down` 不一致 | 直接引用语义令牌，禁止新定义红绿值；加字形双编码 |
| 新增 `--metals-gold` 破坏令牌护栏 | 仅在 `.metals-layout` 作用域声明，且经对比度校验；CI 扫描裸 hex |
| `selectedMetalId` 重启用导致旧 listener/测试失效 | 复用现有信号，更新 `MetalLayout` 订阅；同步改/补 `MetalTable`/`MetalDetail` 测试 |
| 窄屏表格溢出 | 用卡片化降级 + `minmax`/flex，不依赖隐藏列 |
| 色觉障碍 / 屏幕阅读器 | 涨跌用 `aria-label="上涨 x%"` + 字形；详情面板语义化 `section`/`h2` |
| 引入 `CandlestickChart` 体积/性能 | 按需挂载，仅选中品种渲染一图；`prefers-reduced-motion` 关动画 |

### 5.3 工作量粗估（P0+P1）

- 前端：新建 `MetalDetail.jsx`（右栏）、改 `MetalLayout`/`MetalHeader`/`MetalTable`、新增 `market-status` 小组件、响应式 CSS。
- 设计：本分析 + 1 份高保真原型（可下一步产出）。
- 测试：更新 `MetalTable.test.jsx`，新增 `MetalDetail.test.jsx`、`market-status` 测试。
- 预估：约 1 名前端 + 设计 3–5 个工作日（不含 P2）。

## 7. 交易 / 持仓能力已取消（范围收敛为纯行情数据看板）

> **决策变更**：用户在评审原型后明确——**不需要交易下单相关操作，也不需要任何持仓/记账内容，模块只保留纯行情数据看板能力**。因此本节把原 §5 的 P2「真实交易」、本地持仓记账、以及本节全部券商 / 下单 / 持仓内容**整体作废**。

### 7.1 移除范围（已从原型与交付规范删除）
- 下单票（Order Ticket：方向 / 订单类型 / 价格数量 / 实时校验 / 风险揭示 / 提交）。
- 订单状态机与「交易记录」列表（pending→submitted→partial→filled 等）。
- 券商接入层与 `tradingApi` 契约（connect / getAccount / getPositions / placeOrder / cancelOrder / onOrderUpdate）。
- 沙箱 / 模拟环境横幅与「交易中 / 模拟」徽标（改用「实时行情」中性标识 +「示例数据」提示）。
- 「我的持仓（本地记账）」展示区整体移除（模块收敛为纯行情数据看板，不含任何持仓/记账内容）。
- 现状的「录入持仓」`AddMetalModal` 入口一并移除。

### 7.2 保留的看板能力
