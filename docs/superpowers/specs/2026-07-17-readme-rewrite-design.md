# Pulse README 与用户功能手册重写设计

## 背景

当前 `README.md` 仍将 Pulse 主要描述为应用版本检查器，包含旧安装包版本、旧导航结构、已移除的 AI Provider，以及不存在的截图链接。`docs/user-features-guide.md` 同样停留在旧版导航：IT 新闻与微博热搜、基金与贵金属仍被描述为独立一级模块，且未覆盖 GitHub 收录、游戏优惠、首页等当前能力。

Pulse 当前已发展为面向 macOS 与 Windows 的个人桌面信息中心。文档需要准确表达这一定位，同时避免根 README 再次膨胀为难以维护的完整手册。

## 目标

- 将 Pulse 定位为“以应用更新管理为核心的个人桌面信息中心”。
- 让普通用户能在根 README 中快速理解产品、下载安装并找到详细用法。
- 重写完整用户手册，使其与当前导航、功能入口和设置结构一致。
- 采用常青写法，避免硬编码版本号和容易过期的 `v2.x+` 标签。
- 删除失效资源、旧功能描述和过度深入的实现细节。
- 保持 README 与用户手册的术语、模块名称和平台说明一致。

## 受众与文档分工

### `README.md`

主要面向第一次接触 Pulse 的普通用户，也为开发者保留最短的项目入口。目标篇幅约 120–180 行，承担产品首页和快速上手职责。

README 回答以下问题：

1. Pulse 是什么？
2. Pulse 能解决什么问题？
3. 如何下载和安装？
4. 用户数据保存在哪里？
5. 去哪里查看完整用法？
6. 如何开始本地开发？

README 不承担完整功能手册、架构说明或配置字段参考职责。

### `docs/user-features-guide.md`

面向已安装 Pulse、需要了解具体功能和操作方式的用户。它按当前产品信息架构完整描述各一级模块、主要子模块、全局能力、快捷键、设置、数据与常见限制。

用户手册解释“用户能做什么、从哪里进入、操作会产生什么结果”，不展开 IPC、Signals、Worker Pool 等内部实现。

## `README.md` 结构

### 1. 标题与定位

以一句话说明 Pulse 是面向 macOS 与 Windows 的个人桌面信息中心：以应用版本检查和升级为核心，同时聚合资讯、投资行情、游戏优惠、GitHub 项目与 AI 用量。

### 2. 核心能力

用简短列表介绍首页及七个一级业务模块：

- 首页：汇总模块状态、未读角标与快捷入口。
- 新闻：整合 IT 资讯与微博热搜。
- 世界杯：查看赛程、球队、进球榜与淘汰赛对阵。
- 投资：统一管理基金、贵金属与选股分析。
- AI 用量：查看 MiniMax 与 GLM 编程套餐配额和趋势。
- 版本检查：检测应用更新并通过 Homebrew 或 winget 升级。
- GitHub 收录：管理开源项目订阅与 Release 跟踪。
- 游戏优惠：聚合多个平台的折扣、免费活动和热门榜。

另用一组短列表概括全局搜索、提醒、关注列表、最近活动、AI 任务、每日早报、主题切换和自更新。README 不描述具体按钮、字段或子页面细节。

### 3. 下载与安装

- 统一链接到 GitHub Releases。
- 说明 macOS 提供 Apple Silicon 与 Intel 架构安装包。
- 说明 Windows 当前提供 x64 安装包；不承诺尚未进入构建与发布流程的 arm64 原生包。
- 不在文件名中硬编码当前版本号。
- 说明应用升级在 macOS 依赖 Homebrew，在 Windows 依赖 winget。
- 单独说明 Pulse 自身支持检查和下载更新。

### 4. 快速开始与完整指南

提供最短首次使用路径，并链接到 `docs/user-features-guide.md`。具体模块操作、快捷键和设置说明只在用户手册中维护。

### 5. 隐私与本地数据

- 说明主要用户数据保存在本地。
- 说明 AI 密钥通过 Electron `safeStorage` 保护。
- 给出 macOS 与 Windows 的应用数据目录。
- 明确删除 `state.json` 会重置本地状态。
- 不宣称完全离线；新闻、行情、版本、游戏和 AI 等能力会访问对应外部服务。

### 6. 开发

只保留以下核心命令：

- `npm install`
- `npm run dev`
- `npm test`
- `npm run build:mac`
- `npm run build:win`

用一行概括 Electron、Preact、Signals、Vitest 和 Playwright 技术栈。详细架构、Detector、配置和历史设计下沉到 `docs/`。

### 7. 限制与许可证

- Linux 不受支持。
- 部分能力依赖外部服务、Homebrew 或 winget。
- 保留 MIT License。

## `docs/user-features-guide.md` 结构

### 1. 产品与界面概览

说明首页、侧边导航、主内容区和全局弹窗之间的关系。介绍首页状态摘要、未读角标、收藏、拖拽排序、键盘导航，以及侧边导航折叠、排序、隐藏和恢复能力。

### 2. 首页

说明首页卡片、最近访问落点、收藏与排序。记录 `⌘/Ctrl+1–6`（打开当前排序中的前六张卡片）、方向键、`Enter`、空格、`Home`、`End` 等实际已实现的操作，但不写固定卡片总数。

### 3. 新闻

将 IT 资讯和微博热搜描述为同一一级导航下的两个子模块，覆盖刷新、搜索、收藏、已读状态、未读角标及旧数据兜底。

### 4. 世界杯

覆盖赛程、球队、进球榜和淘汰赛对阵，以及比分刷新、比赛搜索、球队阵容和比赛提醒。

### 5. 投资

将基金、贵金属和选股描述为统一投资模块：

- 基金：持仓、实时盈亏、历史记录和提醒。
- 贵金属：实时行情、汇率换算、品种详情和关注。
- 选股：筛选、比较池、个股分析和 AI 诊断。

`⌘⇧F` 与 `⌘⇧M` 分别描述为进入投资页的基金和贵金属子模块。

### 6. AI 用量

介绍 MiniMax 与 GLM 的配额、滚动窗口、趋势、预计耗尽和告警。DeepSeek 不属于用量监控模块，只在 AI 内容生成配置中出现。

### 7. 版本检查

覆盖列表与卡片视图、状态与分类筛选、搜索、支持项的单项升级、按平台执行的批量升级、右键静音、变更日志、诊断、自更新与设置入口，并说明 macOS 和 Windows 的升级依赖。已退役且当前没有用户入口的延后、版本历史、行级回滚和应用行关注不写入手册。

### 8. GitHub 收录

覆盖项目收录、订阅、Release 跟踪、README 解析、自动检查和可选 GitHub Token 配置。

### 9. 游戏优惠

覆盖 Steam、Epic、Xbox、PlayStation、Switch 等平台，并明确区分折扣力度、免费活动和热门榜。免费活动不得与普通折扣混写或暗示所有活动均可永久入库。

### 10. 全局功能

集中介绍全局搜索、提醒、最近活动、关注列表、AI 任务、每日早报、Release Notes、自更新、主题切换和数据导入导出。所有已确认且无冲突的全局快捷键统一汇总，避免散落、重复或宣传当前无法稳定触发的入口。

### 11. 设置与数据

按常规、GitHub、游戏、AI 配置四个设置页签组织。AI 内容生成 Provider 只列当前存在的 DeepSeek、MiniMax 与 GLM。说明 `safeStorage`、本地数据位置、备份、导入和重置方式。

### 12. 故障排查与限制

覆盖应用检测失败、外部 API 不可用、Homebrew 或 winget 缺失、AI 密钥错误、网络超时、数据恢复提示和 Linux 不支持。内部架构问题链接到更深入的项目文档，不在用户手册中展开。

## 事实来源与维护规则

文档事实按以下优先级核验：

1. 当前实际代码和导航配置。
2. `package.json` 中的平台、脚本和构建配置。
3. 当前测试所体现的用户行为。
4. 现有文档仅作为参考，不直接沿用未经代码确认的描述。

若某项能力在代码中存在，但用户入口、状态或行为无法确认，则省略，不推测或写成承诺。

两份文档必须统一使用以下一级模块名称：首页、新闻、世界杯、投资、AI 用量、版本检查、GitHub 收录、游戏优惠。后续新增模块时，应同时检查 README 的能力概览和用户手册的导航章节。

## 实施边界

- 只重写 `README.md` 和 `docs/user-features-guide.md`。
- 不修改应用代码、配置、依赖或测试。
- 不新增或补拍截图。
- 不修改当前工作区中的其他未提交文件。
- 不追溯更新历史规格、计划与 Release Notes。
- 不因本次文档工作运行完整应用测试。

## 验收标准

- `README.md` 约 120–180 行，能快速说明产品定位、核心能力、下载安装、隐私、本地数据和开发入口。
- `docs/user-features-guide.md` 覆盖当前所有一级模块、主要子模块、全局能力、快捷键、设置和数据说明。
- 两份文档不出现已移除的 OpenAI、Anthropic 配置。
- 两份文档不把 IT 资讯、微博热搜、基金或贵金属描述为独立一级导航。
- 不引用缺失截图或 `docs/screenshots/CAPTURE.md`。
- 不硬编码旧安装包版本或使用容易失效的功能版本标签。
- 所有新增或保留的相对链接指向现有文件。
- Markdown 无空章节、`TODO`、`TBD` 或待补占位。
- 若仓库存在 Markdown 或链接检查，则运行该检查；否则通过全文搜索、链接存在性检查和两份文档交叉核对完成验证。
