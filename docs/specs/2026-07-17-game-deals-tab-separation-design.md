# 游戏优惠标签与喜+1隔离设计

## 目标

- 删除平台栏中的“全部”标签，进入游戏优惠页面时默认选择 Steam。
- “折扣力度”只展示付费折扣，不展示喜+1条目。
- 喜+1条目继续集中在“喜 +1 免费领”标签。
- “热门 Top10”保持现有行为，允许出现喜+1条目。

## 设计

渲染层从平台列表中移除 `all`，并将默认平台由 `all` 改为 `steam`。平台切换、筛选和刷新流程保持不变。

聚合层保留 `platform=all` 能力，避免影响后台喜+1检查或其他内部调用。在 `mode=deals` 分支显式排除 `isFree` 条目，再执行折扣门槛过滤和排序。`mode=free` 与 `mode=top` 的行为不变。

## 错误与兼容性

现有外部数据源可能把零元促销返回为普通折扣，因此不能只依赖各数据源的请求参数；聚合层的最终过滤作为统一保障。已有内部调用即使继续传入 `platform=all`，也仍能正常工作。

## 验证

- 渲染层测试确认平台列表不含“全部”，默认平台为 Steam。
- 聚合层测试构造免费条目，确认 `mode=deals` 不返回该条目。
- 现有喜+1与 Top10 测试继续通过，确认两者行为未被误改。
